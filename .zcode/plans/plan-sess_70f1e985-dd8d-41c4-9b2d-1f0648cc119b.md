## Nav rail with 7 view-switching icons

Add a thin vertical icon rail on the far left of the app, below the Masthead. Selecting an icon switches the main content area; the `editor` view reproduces today's Sidebar + two-panel workspace exactly, the other six are lightweight placeholders (stub scope, per your choice). No new deps, state-driven view switching, all code stays in `App.tsx`/`App.css` to match the current single-file convention.

### 1. View model (`App.tsx`, inside `App()` ~line 705)

- Add `type View = "about" | "editor" | "stats" | "globe" | "keys" | "relays" | "models";`
- Add `const [activeView, setActiveView] = useState<View>("editor");` (default `editor` so the app opens exactly as it does today).
- No router. View switching is this one piece of state, passed to a `<NavRail>` and consumed in the render tree.

### 2. `NavRail` component (inline in `App.tsx`, matches the single-file convention)

- Two groups: a top group (flex-grows to push the bottom group down) and a bottom group pinned to the rail's bottom.
- Each item is a `<button>` with the emoji as its content, an `aria-label`, and a `title` tooltip. Active item gets `className="rail-item active"`.
- Icon + target mapping (top → bottom):

  **Top group**
  - 👁️💬 → `about` (About page) — the eye-in-speech-bubble, topmost, per your spec
  - ✏️ → `editor` (current two-panel workspace)
  - 📊 → `stats` (aggregates for the relay)
  - 🌐 → `globe` (future maplibre spherical globe)

  **Bottom group**
  - 🔑 → `keys` (nostr keypair / signer management)
  - 📡 → `relays` (read/write relay management)
  - 🧠 → `models` (LLM keys + inject management)

  (Emoji chosen for the about icon per your instruction, and used throughout for consistency + lightweight stub scope. Trivial to swap to inline SVGs later to match the existing `FolderIcon`/`FileIcon` style.)

### 4. `ViewPlaceholder` component (inline)

A small component that renders a centered title + one-line description of the view's intended purpose + a muted "Coming soon" note. One entry per view in a small lookup object (label + description). Keeps the six stubs visually consistent and gives a clear hook to flesh out later (e.g. the globe stub is where maplibre will land).
